# Hugging Face Dataset Management Scripts

Simplified workflow: **hf-dataset/ IS your media folder**. No more syncing between folders!

## Prerequisites

1. **Git** installed and in PATH
   - Download: https://git-scm.com/download/win
   - Verify: `git --version`

2. **Git LFS** (Large File Storage) installed
   - Download: https://git-lfs.github.com/
   - Or: `winget install GitHub.GitLFS`
   - Verify: `git lfs version`

3. **Hugging Face Authentication**
   ```bash
   git config --global credential.helper manager
   ```
   On first push, enter:
   - Username: Your HF username
   - Password: Your HF Access Token (from https://huggingface.co/settings/tokens)

## Scripts Overview

### 1. `hf-clone.bat` - Clone Repository (Run Once)

**Purpose:** Clone the Hugging Face dataset repository - this becomes your media folder

**Usage:**
```bash
cd mp3-management
hf-clone.bat
```

**What it does:**
- Checks Git and Git LFS are installed
- Clones `dhanyakumarjain/temp` to `../hf-dataset/`
- Configures Git LFS to track `*.mp3` files
- Sets up authentication

**Run this:** Once when setting up for the first time

---

### 2. `hf-push.bat` - Push to Hugging Face

**Purpose:** Upload all files from `hf-dataset/` to Hugging Face (force push)

**Usage:**
```bash
cd mp3-management
hf-push.bat
```

**What it does:**
1. Stages all changes in `hf-dataset/`
2. Commits with your message
3. **Force pushes** to Hugging Face (overwrites remote)
4. Uses Git LFS for large MP3 files

**⚠️ Warning:** Force push overwrites remote - local is source of truth

**Run this:** Whenever you add/update/delete MP3 files in `hf-dataset/`

---

### 3. `hf-reset.bat` - Reset Dataset

**Purpose:** Delete ALL files from Hugging Face dataset

**Usage:**
```bash
cd mp3-management
hf-reset.bat
```

**What it does:**
1. Requires typing "DELETE ALL" to confirm
2. Deletes all files from `hf-dataset/`
3. Force pushes the deletion to Hugging Face

**⚠️ DANGER:** This is destructive and cannot be undone!

**Run this:** Only when you want to completely clear the dataset

---

### 4. `sync-media.bat` - Update Website Data

**Purpose:** Fetch songs from Hugging Face and update `songs.json` and `playlists.json`

**Usage:**
```bash
cd mp3-management
sync-media.bat
```

**What it does:**
- Fetches file list from Hugging Face API
- Generates `data/songs.json` with all songs
- Generates `data/playlists.json` based on folder structure
- Updates `lastUpdated` timestamp in `config.json`

**Run this:** After pushing to HF, before deploying website

---

## Common Workflows

### Initial Setup
```bash
# 1. Clone the HF repository (becomes your media folder)
hf-clone.bat

# 2. Update website data
sync-media.bat

# 3. Deploy website
git-push.bat
```

### Add New Songs
```bash
# 1. Add MP3 files to hf-dataset/ folder
#    Organize in subfolders (each folder = one playlist)

# 2. Push to Hugging Face
hf-push.bat

# 3. Update website data
sync-media.bat

# 4. Deploy website
git-push.bat
```

### Update Existing Songs
```bash
# 1. Edit/delete files in hf-dataset/ folder

# 2. Push changes (force push)
hf-push.bat

# 3. Update website data
sync-media.bat

# 4. Deploy website
git-push.bat
```

### Sync from Another Machine
```bash
# 1. Clone HF repo (first time only)
hf-clone.bat

# 2. Pull latest changes
cd ../hf-dataset
git pull origin main

# 3. Update website data
cd ../mp3-management
sync-media.bat
```

### Complete Reset
```bash
# 1. Delete everything from HF
hf-reset.bat

# 2. Add new files to hf-dataset/
# 3. Push to HF
hf-push.bat

# 4. Update website data
sync-media.bat
```

## Troubleshooting

### Authentication Issues
```bash
# Store credentials
git config --global credential.helper manager

# Then run hf-push.bat and enter:
# Username: your-hf-username
# Password: hf_xxxxxxxxxxxxx (your token)
```

### Git LFS Issues
```bash
# Reinstall Git LFS
git lfs install

# Track MP3 files
cd ../hf-dataset
git lfs track "*.mp3"
git add .gitattributes
git commit -m "Track MP3 files with LFS"
git push --force origin main
```

### Push Rejected
If push is rejected, use force push (already enabled in hf-push.bat):
```bash
cd ../hf-dataset
git push --force origin main
```

### Large Files Not Uploading
Verify Git LFS is tracking MP3 files:
```bash
cd ../hf-dataset
git lfs ls-files
```

Should show your MP3 files. If not:
```bash
git lfs track "*.mp3"
git add .gitattributes
git add *.mp3
git commit -m "Track with LFS"
git push --force origin main
```

## Important Notes

1. **Single Source of Truth:** `hf-dataset/` is your only media folder
2. **Force Push:** Always uses force push - local overwrites remote
3. **Git LFS Required:** MP3 files must use Git LFS (handled automatically)
4. **No media/ folder:** Old `media/` folder is deprecated and gitignored
5. **Folder = Playlist:** Each subfolder in `hf-dataset/` becomes a playlist
6. **Storage Limits:** Check your Hugging Face plan for storage limits
